*CTF2019-hackme-WP

Jul 8, 2021

首先看一下启动参数

qemu-system-x86_64 \
    -m 256M \
    -nographic \
    -kernel bzImage \
    -append 'console=ttyS0 loglevel=3 oops=panic panic=1 kaslr' \
    -monitor /dev/null \
    -initrd initramfs.cpio \
    -smp cores=4,threads=2 \
    -cpu qemu64,smep,smap 2>/dev/null

开启了 kaslr 和 smep，smap。

这道题是一个堆上溢出造成的 UAF，具体的，在 0x30002 功能，也就是 edit 功能处
