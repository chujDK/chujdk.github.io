操作系统导论-OSTEP 简单笔记

Posted on Jul 2, 2021

进程介绍 homework

  1. CPU 利用率应该为 100%，因为不存在 IO 操作
  2. 需要 5 + IO 所需的时间片个时间，由于 IO 需要 4 个时间片，所以总共需要 9 个时间片
  3. 需要 max(1 + IO 所需时间片, 5) + 1 = 6 个时间片
  4. 设置 SWTICH_ON_END 后，由于进程不会被阻塞，所以进行 IO 操作的进程会空占 CPU
  5. 设置 SWTICH_ON_END 后，由于进程发生 IO 后会被阻塞，所以进行 IO 操作的进程不会空占 CPU，会进行调度执行另一进程
  6. 第一个进程需要执行 3 条 IO 操作，如果在阻塞被唤醒后不立即换上执行队列头在最后就会单独执行两次 IO 操作资源未被有效利用
  7. 立即执行可以尽可能的在每次阻塞时有别的进程可以执行，但是事实上立即执行还可以尽快释放 IO 设备，让别的进程（如果需要）可以尽早使用此设备

这一章主要综述了进程的一些概念，个人理解进程是现代操作系统的虚拟化和隔离的体现。

进程 API homework

  1. #include <stdio.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/wait.h>

int main()
{
	int x = 0;
	x = 100;
	int rc = fork();
	if (rc == 0)
	{
		// this is the child routine
		printf("child: x = %d; (pid) = %d\n", x, getpid());
		x = 120;
		printf("child: x = %d; (pid) = %d\n", x, getpid());
	}
	else
	{
		// this is the parent routine
		wait(NULL);
		printf("parent: x = %d; (pid) = %d\n", x, getpid());
	}
}
    由于 fork 之后，子进程是父进程的“镜像”，所以 x 的值仍为 100。在子进程或者父进程中修改此变量对另一进程中的此变量必然是不会有影响的，这是很好理解的，本身进程就是隔离资源的，当然不会对别的线程中的资源做出修改。
  2. #include <stdio.h>
#include <unistd.h>
#include <fcntl.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <sys/wait.h>

int main()
{
	int fd = open("./2.txt",O_RDWR | O_CREAT);
	int rc = fork();
	if (rc == 0)
	{
		// this is the child routine
		for (int i = 0; i < 10; i++)
			write(fd, "child\n", 6);
	}
	else
	{
		// this is the parent routine
		for (int i = 0; i < 10; i++)
			write(fd, "parent\n", 7);
	}
}
    同样的，由于子进程为父进程的镜像，打开的文件是共享的，描述符也相同。两者一起写入会造成竞争，最后输出的文件内容不可预测。 3.
